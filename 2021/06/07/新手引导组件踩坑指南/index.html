新手引导组件踩坑指南

发表于2021-06-07|更新于2021-06-07|Vueshepherd.js
|阅读量:

基于 vue-tour、driver.js 和 shepherd.js 对比得出的结论，其他的不做讨论。

vue-tour

GitHub

优点：

  • 支持 vue 指令使用和实例化调用，仅限 Vue 可用
  • 比较轻巧，风格简洁
  • 支持自定义扩展功能

缺点：

  • 不支持黑色遮罩层，不支持高亮目标元素
  • 还需要额外的在 template 里面引用标签

显示效果预览：

点击这里查看官方的🌰

driver.js

GitHub

优点：

  • 无依赖，安装包很小，压缩完 4kb
  • 文档简洁又不缺功能，实例化调用，不局限是 Vue 还是 React 框架

缺点：

  • driver.js 实现方式属于是，修改父级的 z-index 层级实现遮罩，这样会有一个致命问题，当作用的新手引导元素父级存在定位是 fixed 的情况时，目标元素的层级始终无法超过白色高亮遮罩层。

来两张对比图会好理解一些：

点击这里查看官方的🌰

shepherd.js

优点：

  • API 众多，功能强大，多种场景都可以通过提供的 API 配置出来
  • 对比 driver.js，shepherd.js 使用了 svg 的 path，解决了父级出现 fixed 定位时目标元素被遮住的问题

显示效果如下图：

缺点

  • 依赖 popperjs 包比较大
  • API 自定义程度比较高，而且结构比较复杂，建议封装一层再使用。
